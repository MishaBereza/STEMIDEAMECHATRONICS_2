Проект: Adaptive Difficulty Tournament System
Файл документації: оновлений огляд файлів і функціональності (українською)
Дата: 2026-03-13

1) Короткий опис проєкту
- Це монолітний Flask веб-додаток для найму турнірів з командами, відправки рішень, оцінок та адміністративного управління.
- Стек: Python 3.11+, Flask, Flask-SQLAlchemy, SQLite (data.db), Jinja2, pytest.
- Локальна авторизація: простий механізм через `session['user_id']` для користувача та `session['admin']` для адміну.

2) Швидкий старт
1. Створіть та активуйте віртуальне середовище:
   ```powershell
   python -m venv .venv
   .\.venv\Scripts\Activate.ps1
   pip install -r requirements.txt
   ```
2. Запустіть додаток:
   ```powershell
   set FLASK_APP=app.py
   set FLASK_ENV=development
   flask run
   ```
3. Перейдіть у браузері: http://127.0.0.1:5000

3) Усі ключові файли і призначення
- `app.py`: ядро додатка.
  - Маршрути для користувача (`/`, `/register`, `/tournament/<tid>`, `/tournament/<tid>/register`, `/round/<rid>/submit`, `/team/<teamid>`, `/leaderboard/<tid>`).
  - Адмінські маршрути: `/admin`, `/admin/dashboard`, `/admin/users`, `/admin/tournaments`, `/admin/tournament/create`, `/admin/tournament/<tid>/edit`, `/admin/tournament/<tid>/delete`, `/admin/team/<teamid>/delete`, `/admin/team/<teamid>/decide`, `/admin/change_password`, `/admin/logout`.
  - Механізм генерації/читання пароля з `admin_password.txt` через `get_or_create_admin_password()`.
  - Контекстні процесори: `get_current_user` та `t(key)` для перекладів (UA/EN) з `translations.py`.
  - Функція `create_tables()` створює БД таблиці при першому запиті.

- `models.py`: визначає сутності.
  - `User`: id, first_name, last_name, email(unique), age, bio, role (admin/team/jury).
  - `Tournament`: id, name, description, max_teams, status (default "Draft"), relazione `rounds`.
  - `Round`: id, tournament_id, title, description, must_have, start_at, end_at, level, status, relación `submissions`.
  - `Team`: id, name, captain_id→User, `members` (many-to-many через `team_members`), tournament_id, submissions, repo_url, live_url, comments, submitted_at, submission_status.
  - `Submission`: id, team_id, round_id, repo_url, demo_url, description, submitted_at (UTC), evaluations.
  - `Evaluation`: id, submission_id, jury_id, score_tech/func/ui, comment, метод `total`.

- `utils.py`:
  - `calculate_average_score(scores)` — середнє, або 0.
  - `adjust_difficulty(current_level, average_score)` — адаптивне підвищення/пониження рівня (avg>80 +1, avg<50 та level>1 -1).
  - `generate_new_round(current_level, scores)` — обчислює рівень та повідомлення про наступний раунд.

- `translations.py`:
  - Зберігає словники `TRANSLATIONS` та функцію `get_text(key, lang)` для локалізації.

- `demo_setup.py`:
  - Скрипт з демо-даними: admin, jury, користувачі, турнір, раунд, команда. Запускається в app context. (Є невелика неточність: `Team` не імпортовано, але логіка ідея цього скрипту).

- `templates/`: Jinja2-шаблони UI.
  - `base.html`, `index.html`, `tournament.html`, `register.html`, `register_team.html`, `team.html`, `submit.html`, `profile.html`, `leaderboard.html`, `admin_login.html`, `admin_dashboard.html`, `admin_users.html`, `admin_tournaments.html`, `admin_create_tournament.html`, `admin_edit_tournament.html`, `admin_teams.html`, `admin_change_password.html`, `evaluate.html`.

- `tests/`:
  - `test_models.py`, `test_registration.py`, `test_team_submission.py`, `test_round_submission.py`, `test_admin_submission.py`, `test_utils.py`, `test_deletion.py`, `test_admin_delete_user_team.py`.

- `admin_password.txt`: зберігає пароль для входу в адмінку (за умовчанням "admin123" при першому запуску).

4) Важливі нюанси логіки
- Авторизація:
  - `session['user_id']` і `session['admin']`.
  - `admin_required` декоратор для захисту адмінських маршрутів.

- Реєстрація користувача:
  - `/register` POST: унікальний email, унікальне поєднання first+last.
  - `role` встановлюється `team` за замовчуванням.

- Реєстрація команди:
  - `/tournament/<tid>/register` POST: captain Email має існувати.
  - Заборонено дублікати учасників у одному турнірі.
  - Учасники додаються з уже зареєстрованих користувачів.

- Подання рішення:
  - `/round/<rid>/submit`:
    - доступно, коли турнір `status` в (`Submission`, `Running`).
    - команда вибирається з тих, до яких належить користувач (або всі для адміна);
    - submitter перевіряється по email (капітан/учасник) або адмін.
    - обов’язкове `repo_url`.

- Сторінка команди:
  - `/team/<teamid>` GET/POST: капітан (або адмін) може редагувати репозиторій, live URL, коментарі, склад учасників.
  - Кнопка `send` встановлює `submission_status='Pending'` та `submitted_at`
  - Якщо статус турніру `Finished` і `submission_status` пустий — автоматично `Submitted`.

- Лідерборд:
  - `/leaderboard/<tid>`: для кожної команди обчислюється середній бал всіх оцінок (Evaluation.total) по її поданнях.

- Адмінка:
  - `/admin` логін за паролем з `admin_password.txt`.
  - `/admin/dashboard` — статистика counts.
  - `/admin/users`, `/admin/user/<uid>/delete` (не можна видаляти admin, капітанів без вилучення команди).
  - `/admin/tournaments`, створення, редагування, видалення.
  - `/admin/tournament/<tid>/teams` — перегляд команд.
  - `/admin/team/<teamid>/delete` — з очищенням асоціацій, submissions, evaluations.
  - `/admin/team/<teamid>/decide` — Accept/Reject/Return.
  - `/admin/change_password` — зміна пароля адміна.

5) Поточні обмеження і рекомендації
- Тестування: запустіть `python -m pytest`.
- Безпека: замінити власну сесійну авторизацію на Flask-Login + хешування паролів, облік прав доступу `role`.
- Міграції: додати Flask-Migrate/Alembic замість ручної `db.create_all()`.

6) Оновлення від попереднього варіанту
- У документації додано реальні URL-адреси та правила, перевірку статусів, облік `Round` і `Team` взаємодії.
- Видалено некоректні згадки про відсутні `/create_demo_data` роут, `admin_console.bat` (необов’язково), та вказано фактичні відсутні/присутні поля.
- Підкреслено, що `demo_setup.py` має помилку імпорту `Team` (треба виправити).
